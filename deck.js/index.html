What is Riak?

Riak is an open source, highly scalable, fault-tolerant distributed database. Different than traditional relational databases and other 'NoSQL' non-relational databases

Basho

Features

  • Key Value store
  • Highly available, the AP in CAP
  • Clustered by default
  • Map Reduce processing
  • Eventually consistent
  • Secondary Indexes
  • Full text search
  • Written in Erlang!

Background

Based on the Amazon's Dynamo Key Value store and Dr Eric Brewer's CAP Theorem.

CAP theorem, states that it is impossible for a distributed computer system to simultaneously provide all three of the following guarantees:

  1. Consistency
  2. Availability
  3. Partition Tolerance

Riak was originally developed by Basho as part of a Salesforce application, but proved to be more popular.

Buckets and Keys

Buckets + keys are the only way to organise user data. All user data is stored and referenced by bucket/key pair.

Bucket is a collection of keys, with associated configuration data.

Keys can be either a string or binary, user defined or riak can generate one.
eg 37458abc752f8413e, fancy_string or 0xDEADBEEF

Links and Metadata

Links are metadata that establish one-way relationships between objects in Riak. They can be used to loosely model graph like relationships between objects in Riak.

Objects can have user defined metadata in addition to the actual object contents, which are exposed as HTTP headers or associative arrays.

eg if /riak/list/2 is part of a linked list.

Clustering

Riak's default mode of operation is in a cluster.

The extent to which data is replicated, and when, and with what merge strategy and failure model, is configurable at runtime.

Riak computes a 160-bit binary hash of the bucket/key pair, and maps this value to a position on an ordered "ring" of all such values.

eg 32 partitions / 4 nodes = 8 vnodes for each node

More Clustering

Writes are split across multiple nodes and only returns success if write is confirmed by N nodes, where N is configurable.

Reads are also split across multiple nodes with the same consensus protocol. Inconsistent read results can be repaired without user intervention (Read Repair).

Ring state is shared around the cluster by means of a gossip protocol.

One is good, Many is Better

Replication is fundamental and automatic in Riak.

All data stored in Riak will be replicated to a number of nodes in the cluster according to the n_val property set on the bucket.

By default, Riak chooses an n_val of 3 for you.

Tune n_val wisely, high values provide more redundency and lower values provide more speed.

Map Reduce

Riak's Map Reduce allows richer queries over the data stored in Riak.

Can be written in Javascript or Erlang.

Spreads the processing of the query to where the data is.

// Map Function function(v) { var words = v.values[0].data.toLowerCase().match('\w*','g'); var counts = []; for(var word in words) if (words[word] != '') { var count = {}; count[words[word]] = 1; counts.push(count); } return counts; }

Map Reduce Cont.

// Reduce Function function(values) { var result = {}; for (var value in values) { for(var word in values[value]) { if (word in result) result[word] += values[value][word]; else result[word] = values[value][word]; } } return [result]; }

Where's the Ruby dude?

Basho provides great Ruby support via the Ripple library, https://github.com/seancribbs/ripple

It includes two namespaces:

  • Riak basic wrapper around typical operations.
  • Ripple provides a rich modeling layer for Riak, based on ActiveModel abstraction similar to ActiveRecord, DataMapper and MongoMapper.

Supports Rails 3, 3.1 and 3.2 (almost).

Saving data to Riak

require 'riak'

# Create a client connection
client = Riak::Client.new

# Retrieve a bucket
bucket = client.bucket("books")

# Create a new object
new_book = Riak::RObject.new(bucket, 'erlang_otp_in_action')
new_book.content_type = 'application/json'
new_book.data = { title: 'Erlang and OTP in Action',
                  author: 'Logan Martin',
                  summary: 'Erlang is an adaptable and fault tolerant functional programming language originally designed for the unique demands of the telecom industry'}

# Save!
new_book.store

Using Ripple ActiveModel

require 'ripple'

class Book
  include Ripple::Document

  property :title, String, :presence => true
  property :author, String, :presence => true
  property :summary, String, :presence => true
end

book = Book.new
book.title = 'The Well-Grounded Rubyist'
book.author = 'David Black'
book.summary = 'The Well-Grounded Rubyist takes you from interested novice to proficient practitioner.'

book.save

results = Riak::MapReduce.new(client).add("books").
  map("function(v) {
     var results = [];
     var book = JSON.parse(v.values[0].data);
     if (book.author == 'Joe Armstrong') {
       results.push(book);
     }
     return results;
  }", :keep => true).run
